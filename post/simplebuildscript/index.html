Continuous Delivery for personal projects

If you are like me, you build side projects from time to time. Some of them are application running on a server. This led me to the question what is a lightweight and simple way to deploy those application. Do I need to setup a full CI/CD pipeline? These are often heavy tools to run production grade software. Maybe a simple script is good enough for my use-case? After all, I’m most likely to abandon this project after a while.

What do I want to achieve? Basically I want to git push and see my changes deployed. Knowing about Git Hooks the post-receive hook looks like what I’ve been searching for. What needs the script to do?

Four Simple Steps

  1. Get the changes
  2. Build a new version with the changes
  3. Shutdown the running application
  4. Start the new build

And this is what I came up with:

#!/bin/sh
cd ../
echo "================================================================"
echo "MERGE CHANGES"
GIT='git --git-dir='$PWD'/.git --work-tree '$PWD''
$GIT merge main
echo "================================================================"
echo "BUILD INITIALIZED"
echo "START BUILD PROCESS"
echo ""
clj -T:build uber

echo "================================================================"
echo "SEARCH FOR RUNNING APPLICATION..."
pid=`ps aux | grep myapp.jar | sed '/grep/d' | awk '{print $2}'`
if [ -n $pid ]
then
    echo "RUNNING APPLICATION FOUND"
    echo "INITALIZE SHUTDOWN"
    kill $pid
    echo "WAIT FOR TERMINATION"
    sleep 3
    pid2=`ps aux | grep myapp.jar | sed '/grep/d' | awk '{print $2}'`
    if  [ -n $pid2 ]
    then
        echo "WAITED 3SEC FOR GRACEFUL SHUTDOWN"
        echo "KILLING PROCESS NOW"
        kpid=`kill -9 $pid2`
        wait $kpid
    fi
    echo "PROCESS IS SHUTDOWN"
fi

echo ""
echo "================================================================"
echo "DEPLOY NEW BUILD"
java -jar target/myapp.jar &

admittedly there is a lot of room for improvements, but it’s good enough for me currently.

How do we set this up?

ssh admin@serverip                  # login to the server
mkdir /var/www/myapp                # create the app dir
cd /var/www/myapp
git init
git checkout release                # we can't push to a branch we have currently checked out

on our local machine

# assuming we are inside our local project dir

git remote add live ssh://admin@serverip:/var/www/myapp
git push live main

on the server

git merge main
cp hooks/post-receive .git/hooks/   # copy our deploy file to the hook dir

next time we push with git push live main our application gets automatically deployed.
