Duality between data and code

The first sentence on the Wikipedia page about “Code as data”:

In computer science, the expressions code as data and data as code refer to the duality between code and data, that allows computers to treat instructions in a programming language as data handled by a running program. [0]

I strongly associate the phrase “data as code” with LISP and I absolutely do love to speak about LISP but this time I will try to explore the other way around: “code as data”.

In functional programming we emphasize programming with pure functions. A pure function, given the same input, will always produce the same output without altering any state. Given that we can put the input x and the computed value y of a pure function fn into a map m = {x y}. Next time someone calls fn(x) we just go to our lookup. When we are able to do this for all possible inputs we are than able to substitute the calculation with a lookup in m. Would that be data as code?

[0] https://en.wikipedia.org/wiki/Code_as_data
